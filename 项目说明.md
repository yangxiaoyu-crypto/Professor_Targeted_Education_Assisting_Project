# 教学智能助手项目说明

## 项目概述

**项目名称**：教学智能助手 (Teaching AI Assistant)

**项目类型**：Web应用程序

**目标用户**：大学教授和教师

**核心价值**：帮助大学教师高效完成高质量的教学设计

## 项目背景

根据项目需求文档，本项目旨在解决以下问题：

### 面临的挑战
1. **新手教师**：从科研转教学，缺乏教学设计培训
2. **课程大纲撰写**：需要符合规范，体现教学理论
3. **教学方法选择**：不了解现代参与式学习方法
4. **评价体系设计**：难以设计科学、公平的评估方案
5. **课程思政融入**：不知如何自然融入思政元素
6. **方案对比优化**：缺乏专业的评价标准

### 对标产品
- **eduaide.ai**：国外知名教学AI助手
- **进阶需求**：
  - 支持教师输入自己的想法并优化
  - 提供多种方案供选择（如6种参与式学习方法）
  - 能够比较不同方案的优劣
  - 生成更规范的课程大纲

## 核心功能实现

### 1. 课程大纲智能生成 ✅
**基于理论**：
- 泰勒原理（目标-内容-方法-评价）
- 逆向设计（UbD - Understanding by Design）
- 布卢姆教育目标分类学
- 加涅九大教学事件

**功能特点**：
- 支持教师输入自己的想法
- AI基于理论生成专业大纲
- 包含7大部分：概述、学习成果、教学内容、教学方法、评价考核、教学资源、课程思政
- 支持流式生成，实时显示
- 可复制、下载

### 2. 参与式学习设计 ✅
**提供6种教学方法**：
1. BOPPPS教学模式
2. 项目化学习（PBL）
3. 翻转课堂
4. 案例教学法
5. 同伴教学法
6. 工作坊模式

**功能特点**：
- 每种方法都有详细的理论说明
- 可根据课程特点选择合适方法
- AI生成具体的教学活动方案
- 包含时间分配、教师话术、学生活动等细节

### 3. 学习目标智能撰写 ✅
**理论基础**：
- 布卢姆认知分类（6个层次）
- 加涅学习结果分类（5种类型）
- SMART原则

**功能特点**：
- 提供布卢姆分类参考表
- 使用准确的行为动词
- 生成可测量、可观察的目标
- 覆盖知识、能力、素质三个维度

### 4. 学生评估方案设计 ✅
**评价理论**：
- 形成性评价
- 总结性评价
- 评价的公平性原则
- 有效反馈的七个原则

**功能特点**：
- 灵活配置形成性/总结性评价权重
- 生成评价量表（Rubric）
- 防止不公平评价的措施
- 建立有效的反馈机制

### 5. 课程思政设计 ✅
**核心原则**：
- 自然融入，不生硬
- 专业特色，有深度
- 学生中心，有共鸣
- 润物细无声

**功能特点**：
- 提供10种思政元素供选择
- 挖掘专业知识中的思政点
- 设计自然的融入方式
- 避免"两张皮"现象

### 6. 课程方案对比优化 ✅
**评价维度**（6维度，100分制）：
- 学习目标设计（20%）
- 内容组织设计（20%）
- 教学方法策略（20%）
- 评价考核体系（20%）
- 创新性（10%）
- 可操作性（10%）

**功能特点**：
- 对比教师方案和AI方案
- 详细的优劣势分析
- 综合评分和可视化
- 针对性改进建议
- 综合优化方案

## 使用场景

### 场景1：新教师备课
**用户**：刚入职的青年教师
**需求**：首次开设《数据结构》课程，需要撰写教学大纲
**使用流程**：
1. 填写课程基本信息
2. 选择BOPPPS教学模式
3. 生成完整课程大纲
4. 设计具体的参与式学习活动
5. 制定评估方案

### 场景2：老教师改革
**用户**：有10年教学经验的教授
**需求**：改进现有课程，提升教学质量
**使用流程**：
1. 输入现有教学设计
2. 使用AI生成优化方案
3. 对比分析两个方案
4. 根据建议改进
5. 融入课程思政元素

### 场景3：教学创新
**用户**：追求教学创新的教师
**需求**：尝试新的教学方法
**使用流程**：
1. 浏览6种参与式学习方法
2. 选择PBL项目化学习
3. 生成详细的PBL方案
4. 设计项目评价体系
5. 实施并收集反馈

### 场景4：课程思政
**用户**：需要融入思政的专业课教师
**需求**：在《人工智能》课程中自然融入思政
**使用流程**：
1. 选择相关思政元素（科学精神、社会责任等）
2. AI挖掘专业知识中的思政点
3. 设计融入方案
4. 在案例、讨论中实施
5. 评价融入效果
